# Git提交安全审查清单

在每次向公共仓库提交内容之前，请务必完成以下检查项，确保信息安全。

## 个人信息脱敏检查

- [ ] 删除或替换真实姓名（使用"用户A"、"测试者"等代替）
- [ ] 删除或替换电话号码（使用"138****0000"等形式）
- [ ] 删除或替换身份证号、护照号等证件信息
- [ ] 删除或替换住址、公司具体地址信息
- [ ] 删除或替换银行账号、支付账号信息
- [ ] 删除或替换邮箱地址中的敏感部分
- [ ] 删除或替换社交媒体账号信息

## 技术信息检查

- [ ] 删除API密钥、访问令牌、密码等机密信息
- [ ] 删除数据库连接字符串中的用户名和密码
- [ ] 删除服务器IP地址和端口信息
- [ ] 删除内部系统架构细节
- [ ] 删除可能暴露技术栈漏洞的信息

## 业务数据检查

- [ ] 删除真实的销售数据、财务数据
- [ ] 删除客户名单和联系信息
- [ ] 删除合同金额和商业条款细节
- [ ] 删除产品成本和定价策略
- [ ] 删除未公开的市场计划

## 文件内容检查

- [ ] 检查截图是否包含敏感信息（打码处理）
- [ ] 检查代码注释是否包含内部信息
- [ ] 检查日志文件是否包含敏感数据
- [ ] 检查配置文件是否包含硬编码凭证

## 提交前操作步骤

1. 使用`.gitignore`文件排除不应提交的文件类型
2. 执行`git diff --cached`检查即将提交的更改
3. 使用`git secrets`或其他扫描工具检查敏感信息
4. 至少一人复查（如果是团队协作）
5. 确认无误后再执行`git commit`

## 紧急处理措施

如果意外提交了敏感信息：

1. 立即撤销提交：`git reset --soft HEAD~1`
2. 移除敏感信息后重新提交
3. 如果已经推送，需要强制推送更新：`git push --force-with-lease`
4. 如果是重要凭证信息，立即更换密码或重新生成密钥
